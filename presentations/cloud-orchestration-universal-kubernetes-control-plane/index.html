Cloud Orchestration

with a Universal Kubernetes Control Plane

A clean starting point for platform teams aligning apps, clusters, policies, and infrastructure.

Niklas Schuster Reveal.js deck skeleton

Problem space

Cloud orchestration is fragmented

Many control planes

Cloud APIs, CI/CD, GitOps, cluster lifecycle tooling, policy engines, and app platforms all compete for ownership.

Platform teams need one intent layer

A universal Kubernetes control plane can expose consistent APIs while delegating execution to specialized controllers.

Direction

Universal control plane primitives

  • Declarative APIs for apps, environments, and cloud resources
  • GitOps reconciliation as the operational backbone
  • Policy, identity, and observability built into the platform contract
  • Composable controllers instead of one monolithic orchestrator
