  • Pipelines are excellent for compiling code and producing deployable artefacts. They are designed for deterministic, one-time execution where a defined sequence of steps produces a predictable output.
  • However infrastructure and platform resources are fundamentally different: they require continuous reconciliation, lifecycle management, and drift correction.

“Simplifying Cross And Multi-cloud Services With Cloud Agnostic Custom Kubernetes Resources”

  • “Simplifying Cross And Multi-cloud Services With Cloud Agnostic Custom Kubernetes Resources”
  • 9
  • Crossplane provides new concepts of Composite Ressorces Definitions (XRD) and Compositions to enable the provisioning and management of custom Kubernetes resources, Crossplane Composite Resources (XR), with a self defined layer of abstraction.
  • Where a XRD describes the API Spec of an XR and a Composition is a blueprint and describes what is created in the underlying cloud provider when a new XR is transferred to the control plane. Each XRD can be backed and filled by multiple compositions.

Simplify, Abstract, Deliver

  • Simplify, Abstract, Deliver
  • Compose Your Own Custom Cloud API
  • 10
  • Crossplane allows a platform team to create custom infrastructure abstractions, defining reusable components and exposing them via a custom API.
  • This hides complexity, not only simplifying resource management for developers through self-service APIs but also abstracting away the challenges of working across different contexts and cloud environments.
  • By using standardized building blocks, teams can seamlessly operate across multiple clouds, maintaining flexibility and scalability without needing deep infrastructure knowledge, all while streamlining operations.
