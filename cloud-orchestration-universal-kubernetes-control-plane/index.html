The CI/CD Anti-Pattern: Pipelines Are Not Platforms

Pipelines produce artifacts. Platforms continuously manage desired state.

Pipelines are excellent for compiling code and producing deployable artifacts. They are designed for deterministic, one-time execution where a defined sequence of steps produces a predictable output.

Infrastructure and platform resources are fundamentally different: they require continuous reconciliation, lifecycle management, and drift correction.

Keep pipelines focused on build and packaging tasks.

“Simplifying Cross And Multi-cloud Services With Cloud Agnostic Custom Kubernetes Resources”

Crossplane introduces composite resources, XRDs, and Compositions.

Crossplane provides Composite Resource Definitions (XRDs) and Compositions to provision and manage custom Kubernetes resources with a self-defined abstraction layer.

An XRD describes the API spec of an XR. A Composition describes what is created in the underlying provider when a new XR reaches the control plane.

Simplify, Abstract, Deliver

Compose Your Own Custom Cloud API

SimplifyAbstractDeliver

Crossplane allows a platform team to create custom infrastructure abstractions, defining reusable components and exposing them via a custom API.

This hides complexity for developers through self-service APIs and abstracts the challenges of working across cloud environments.

Standardized building blocks let teams operate across multiple clouds without requiring deep infrastructure knowledge.
